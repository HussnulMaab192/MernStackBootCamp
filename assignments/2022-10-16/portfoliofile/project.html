  • Charity:Clothe the Poor:-
    • Intrduction:

      Clothe the poor software is a project to connect donors and donees to donate the clothes. The donated clothes are collected from homes and are sent to welfare Organizations or Orphanages. This system is very useful for society as it saves time to search for needy people to donate clothes. It provides a benefit to donors and donees to give and take the clothes without having a visit to different places. The clothe the poor is an online system and also have different warehouses to store the donated clothes. This system also maintains the history of received and delivered clothes.

  • Scope:

    Our goal is to create an application to donate and receive clothes for needy persons through an online system without any difficulty. Our goal is to make a quality donation app that provides quick service for donors to donate clothes to donees. An interface is provided to the donor in which the user can easily donate his clothes and give all information about them e.g.: Size, Category, and Color. The administration manages the clothes according to the size, color, and age. The collecting agent collects the clothes from the donor from its location and delivers that clothes to the donee. Our warehouses will be located in different cities and they can contact one another to get the required stock of clothes.

Signup Screen

Sign up screen on which if user is not registered then it will fill the information and get registered as a donor or donee and information includes Full Name, email address, phone no, password, type (Donor/Donee), gender and if his/her is done then he/she is individual or an organization. After filling this form user clicks on Signup button and then login to move on next screen

There are two text fields, login button and if user not registered then there is a button for Create New Account. One text field is for email and other one is for password. After entering the email and password then user clicks on login button and user moves to next screen. If user not registered then it clicks on Create New Account button and moves to Signup screen.

Donor View Screen

There is Date Time Picker’s button, comment text field and pickup location button for select Pickup location. This is main screen of donor to request for donate cloths. Donor will select date time on which collector will collect from him/her on that particular date. If he/she want to add additional information then they will add that information in comment field and last thing that he/she will add their location from where collector collect cloths from them and then after adding whole information Donor clicks on Donate button then this request is forwarded to the nearest collector who is available in the 10Km of area from donor’s location.

Donee View Screen

There is donee view Screen, there is a radio buttons for gender selection, text field for Color, drop-down for Clothe-type, Size, Age and Condition.

Collector View Screen

There are three buttons one is for Donor Request and second one is for schedule donee requests on the basis of date and other one is for Donee Request.

Admin(warehouse) View Screen

There are six buttons one is for view clothes stock by clicking on this button user see the full stock available in the warehouse. The second one is Registered collector’s button if warehouse want to register new collector then they put their data and store into database. The third button is Request from donor on selection of this button the warehouse will check new donation request from donor. The fourth button show the new requested clothes from donee. Fifth button is use to add more clothes into the stock. And the last one is used to view the schedule notifications request on basis of current date.

  • Health:Patient Health Plateform:-
    • Intrduction:

      Current and older system(s) only collect data which is of no use to government and its officials. Manual registers have a lot of paperwork, which is hard to maintain and monitor by health workers. Patient Health Platform (PHP) system helps replace the immense paperwork of current system and automates the data collection system. PHP system will be used to collect and keep detailed and continuous track of information about health of a person and their family. It minimizes the need of repeating same medical tests for a person. It gives more insight about patient’s medical history to medical care providers. Data gathered is updated in real-time and synchronized to online data repository. Researchers can easily access the statistics of health records of various regions. They can perform analysis on this cumulative data gathered in the system but cannot access personal information of a person or their family. Government and other officials can make decisions based on summary of this data and its reports. This helps in taking certain precautions related to disease spreading. Users can be notified with alerts in case of any pandemics.
